Metadata Alignment

In order to improve the interoperability of data, the metadata has been compared and aligned, using all metadata standards found in the previous step. The metadata has been divided into four dimensions: Who, When, Where, What

WHO

Property Dublin Core MARC21 MODS
Creator/author dcterms:creator 100 <name><namePart>
Publisher dcterms:publisher 260$b <originInfo><publisher>


WHEN

Property Dublin Core MARC21 MODS
Creation/Publication Date dcterms:created, dcterms:available 260$c, 264$c <originInfo><dateIssued>


WHERE

Property Dublin Core MARC21 MODS
Production Location dcterms:coverage, dcterms:spatial 260$a, 264$a <geographic>
Current Location dcterms:coverage, dcterms:spatial 852$a <physicalLocation>


WHAT

Property Dublin Core MARC21 MODS
Subject dcterms:subject 6xx <subject>
Language dcterms:language 041 <language>
Title dcterms:title 240 <title>
Material dcterms:medium 3xx <physicalDescription><form type="material">
Type dcterms:type 008, 336 <typeOfResource>

Theoretical model

The theoretical model represents a way to abstract the relationships between the items. It allows to create a better visualization specifying the Who, the When, the Where and the What of each object. The items have been grouped into three categories, in order to create more specific connections with the adequate metadata: Artefacts, Multimedia and Textual Data (Text). Two visualizations have been provided: one for a general understanding of the properties assigned to each category of items and another with the actual metadata. The Artefacts have metadata about the following properties: Creator, Publisher, Creation/publication date, Production Location, Current Location, Subject, Title, Material and Type. The Texts have metadata about the following properties: Author, Publisher, Creation/publication date, Production Location, Current Location, Subject, Title, Language and Type. The Multimedia items have metadata about the following properties: Creator, Publisher, Creation/publication date, Production Location, Subject, Title, Language and Type. Summary of the differences: Multimedia items don’t have Material and Current Location properties; Texts don’t have the Material property; Artefacts don’t have the Language property

Conceptual model

In our conceptual model, the most general properties are expressed with the widely used ontologies Dublin Core and Schema.org. In order to highlight specific features for our entities and items, we chose FOAF for Persons and Organizations, and SKOS for the main concept of our domain. Furthermore, we implemented FABIO to distinguish different types of textual data, and CIDOC-CRM and DBPEDIA-OWL for the selected artefacts and multimedia.

RDF Production

Following the description of our items, we created an RDF file containing all entities with their personal URI. We provided a meaningful description of our items by expressing their relation to existing entities on the semantic web using Ontologies, Value Vocabularies and Authority Controls-specifically Art and Architecture Thesaurus for concepts and types, Geonames for places and VIAF for authorities.
